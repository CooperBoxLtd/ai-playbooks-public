# CooperBox Playbooks

These playbooks are the detailed reference layer for the concise rules in `../../AGENTS.md`.

Use only the playbooks relevant to the current task:

- `00-core-laws.md`: governing principles for all work.
- `01-codex-operating-model.md`: how Codex should inspect, implement, validate, and report.
- `02-production-stack.md`: production layers and responsibilities.
- `03-security.md`: identity, authorization, tenant isolation, secrets, validation, and AI security.
- `04-data-state.md`: ownership, state, multi-tenancy, schema changes, events, and audit.
- `05-ai-systems.md`: intent, context, model abstraction, validation, evaluation, and AI endpoint controls.
- `06-deployment-recovery.md`: deployment, migrations, jobs, rollback, and recovery.
- `07-observability.md`: logs, metrics, tracing, alerting, and incident readiness.
- `08-cost-control.md`: attribution, budgets, rate limits, caching, provider controls, and AI cost rules.
